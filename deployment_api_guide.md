# 🚀 خطة إطلاق النظام وتجهيز الـ API للمطور (Deployment & API Guide)

أهلاً بك في مرحلة الإطلاق! لرفع النظام إلى الإنترنت وجعل مطور التطبيق (Flutter/Java/Swift) يبدأ العمل، قمت بتجهيز خطة متكاملة ومرتبة كخارطة طريق لك:

---

## 🟢 أولاً: اختيار الاستضافة المناسبة (Hosting)

بما أن النظام الخاص بك عبارة عن لوحة تحكم (Laravel) وتطبيق جوال (يتصل عبر API)، فأنت تحتاج إلى استضافة سريعة وتدعم العمليات الخلفية (Jobs/Queues) الخاصة بالتنبيهات.

### الخيار الأفضل (الموصى به بشدة لحجم مشروعك)

**استضافة VPS (سيرفر افتراضي خاص)** من **Hostinger** أو **DigitalOcean**.

- **السبب:** تعطيك تحكم كامل، سرعة عالية جداً للـ APIs، وسعرها ممتاز للبدايات (تبدأ من 5-10 دولار شهرياً).
- **المطلوب:** تشتري VPS، وتثبت عليه أداة مجانية مثل (Laravel Forge أو CloudPanel أو CyberPanel) لإدارة قاعدة البيانات وتشغيل المشروع بسهولة.

### الخيار للمبتدئين (cPanel Shared Hosting)

**استضافة مشتركة** من **Hostinger** أو **Namecheap**.

- **السبب:** واجهة رسومية سهلة جداً (cPanel)، لا تتطلب خبرة بالسيرفرات.
- **العيوب:** قد تكون أبطأ قليلاً، وأحياناً تواجه قيوداً في تشغيل الـ Cron Jobs (الأوامر المجدولة الخاصة بتنبيهات الغياب).

> **🎯 نصيحتي لك:** ابدأ مع **Hostinger VPS (خطة KVM 1 أو KVM 2)**. سأقوم أنا (الذكاء الاصطناعي) بمساعدتك خطوة بخطوة في تثبيت لوحة التحكم ورفع المشروع عندما تشتريها.

---

## 🟡 ثانياً: تجهيز مشروعك للرفع (Deployment Prep)

قبل أن نرفع الملفات، سنحتاج للقيام بالآتي معاً:

1. شراء **اسم نطاق (Domain Name)** مثل (student-hub.com).
2. تجهيز قاعدة البيانات على السيرفر الجديد.
3. التعديل على ملف `.env`:
   - تغيير `APP_ENV=production` و `APP_DEBUG=false`.
   - تغيير `APP_URL` إلى رابط موقعك الجديد.
   - إعداد بيانات قاعدة البيانات الحقيقية.
4. أخذ نسخة من مجلد مشروعك (ملف ZIP) أو رفعه عبر GitHub (وهو الأفضل والموصى به).

---

## 🔵 ثالثاً: ماذا يحتاج مطور تطبيق الجوال؟ (The API)

مطور التطبيق لا يحتاج كود الموقع، بل يحتاج إلى **(روابط API + توثيق يقرأه ليصمم التطبيق)**.

### وضعنا الحالي في المشروع

حالياً، نحن بنينا النظام على أساس **"لوحة تحكم (Web)"**، وقمنا فقط بتجهيز مسارات واجهة لمسح الـ QR. لكي يعمل التطبيق، **نحتاج نبني الـ API الخاص به**.

### 🛠️ ما الذي يجب أن أقوم (أنا وأنت) ببرمجته الآن قبل تسليم المطور؟

1. **تثبيت Laravel Sanctum:** وهو نظام الحماية الذي سيعطي تطبيق الجوال "مفاتيح توكن - Tokens" لكي يتمكن الطالب من تسجيل الدخول.
2. **بناء مسارات الـ API (API Routes):**
   - مسار لتسجيل الدخول (`/api/login`).
   - جلب جدول محاضرات الطالب (`/api/schedule`).
   - جلب الإشعارات (`/api/notifications`).
   - مسارات الغياب والأعذار، إلخ.
3. **صناعة ملف Postman (API Documentation):**
   - بعد أن نبني الـ APIs، سأقوم بإنشاء ملف بصيغة (JSON) يسمى بـ "Postman Collection".
   - سأعطيك هذا الملف، وأنت تعطيه لمطور التطبيق. سيفتحه في برنامج Postman وسيجد كل الروابط مشروحة مع أمثلة.

---

## 📝 خطة العمل المقترحة لك (Action Plan)

1. **اليوم:** احجز الاستضافة (VPS أو cPanel) والدومين (أنصح بـ Hostinger).
2. **غداً:** اطلب مني بناء الـ `APIs` الخاصة بتطبيق الجوال (نبدأ بنظام تسجيل الدخول وجلب الجدول).
3. **بعد اكتمال الـ APIs:** سأستخرج لك ملف Postman لترسله لمطور التطبيق ليبدأ فوراً.
4. **في نفس الوقت أو الأسبوع القادم:** نقوم معاً برفع المشروع برمجياً إلى الاستضافة الحية، ليكون الرابط الفعلي (الذي سيستخدمه التطبيق) يعمل.

**أخبرني: هل تريد أن نبدأ ببناء وتجهيز الـ APIs لمطور التطبيق الآن؟ أم تريدني أن أشرح لك بالتفصيل خطوة شراء الاستضافة؟**
