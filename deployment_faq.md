# أجوبة لأسئلتك حول الاستضافة ورفع المشروع 🌐

أهلاً بك! أسئلتك ممتازة جداً وهي أسئلة كل مبرمج أو صاحب مشروع يرفع موقعه لأول مرة. سأجيبك عليها بالتفصيل وببساطة:

---

### 1️⃣ كيف أضيف التحديثات بسرعة؟ هل أحتاج لرفع الملفات كل مرة؟

**الجواب: لا، لن تضطر لرفع الملفات يدوياً كل مرة!**
الطريقة الاحترافية (والتي سأعلمك إياها) تسمى **الربط المستمر (CI/CD) عبر GitHub**.

- نقوم برفع الكود الخاص بك إلى مستودع خاص ومخفي على موقع **GitHub**.
- نربط مستودع GitHub بالاستضافة.
- **النتيجة:** عندما نبرمج ميزة جديدة على جهازك، نكتب أمر رفع سريع (Push). الاستضافة ستكتشف التحديث تلقائياً، وتسحب الكود الجديد لوحدها في ثوانٍ دون أن يتوقف موقعك أو تحذف أي ملفات!

### 2️⃣ هل يجب أن يكون الـ API موجوداً في مجلد المشروع قبل الرفع؟

**الجواب: نعم، تماماً.**
مسارات الـ API هي جزء لا يتجزأ من كود المشروع (تحديداً في ملف `routes/api.php` والمتحكمات الخاصة بها). لذلك، كما طلبت، سنقوم أولاً ببناء وتجهيز كل الـ APIs، ثم نرفع المشروع بالكامل (لوحة التحكم + الـ API) مرة واحدة إلى الاستضافة.

### 3️⃣ هل لا بأس إذا استعملت استضافة مشتركة (Shared Hosting)؟ وما الفرق؟ وما الإعدادات؟

**الجواب: لا بأس في البداية، لكن يجب أن تعرف الفروقات:**

- **الاستضافة المشتركة (Shared Hosting - cPanel):**
  - **ما هي؟** كأنك تستأجر "غرفة" في شقة كبيرة مع أشخاص آخرين. تشاركون نفس موارِد السيرفر (الإنترنت، المعالج).
  - **المميزات:** سهلة جداً للمبتدئين لوجود واجهة رسومية (cPanel).
  - **العيوب الجوهرية لمشروعك:**
    - تمنع غالباً الأوامر التي تعمل في الخلفية بكثرة (مثل الجدولة الدقيقة لإرسال إشعارات المنح أو الحرمان للطلاب).
    - صلاحياتك محدودة، وتثبيت وتحديث بيئة Laravel قد يكون متعباً وتظهر فيه أخطاء مسارات.
- **الاستضافة الخاصة (VPS):**
  - **ما هي؟** كأنك تستأجر "شقة كاملة" لك وحدك. لك كامل الموارد والصلاحيات.
  - **المميزات للمعاهد/التطبيقات:** سريعة جداً للـ API، تدعم أوامر إرسال الإشعارات (Queues/Cron) بكل كفاءة، يمكنك تثبيت أحدث إصدارات قواعد البيانات بدون قيود.
  - **الإعدادات:** سنقوم بتثبيت لوحة تحكم مجانية (مثل CloudPanel) عليها بضغطة زر، فتصبح واجهتها سهلة مثل cPanel!

### 4️⃣ كم المساحة التي سيتم إعطائي إياها (أحتاج مساحة للمصادر والمحاضرات)؟

شركات الاستضافة تعطيك مساحات مختلفة حسب الباقة.
- في **Hostinger (الاستضافة المشتركة):** الباقة المتوسطة (Premium) تعطيك **100 جيجابايت** SSD.
- في **Hostinger (VPS):** الباقة الأولى (KVM 1) تعطيك **50 جيجابايت** NVMe (أسرع بكثير)، والباقة الثانية تعطيك **100 جيجابايت**.
- **ملاحظة هامة:** 50 أو 100 جيجابايت تعتبر **رقم ضخم جداً جداً** للبداية لملفات (PDF، صور، ملفات وورد). (الفيديوهات الثقيلة تُرفع عادة على يوتيوب ويرفق رابطها في النظام، لا تُرفع مباشرة على أي استضافة لتوفير المساحة والسرعة).

### 5️⃣ هل أحتاج إلى Domain (نطاق) ولماذا؟

**الجواب: نعم وبقوة!**
- **ما هو الدومين؟** هو اسم موقِعك (مثلاً `your-institute.com`).
- **لماذا؟**
    1. تطبيق الجوال (Flutter أو غيره) يحتاج إلى رابط رئيسي ثابت ليتصل به لجلب البيانات (مثلاً `https://your-institute.com/api/login`). إذا لم يكن لديك دومين، سيتصل التطبيق برقم IP (مثل `192.168.1.5`) وهذا الرقم يتغير ويكون شكله غير احترافي وغير آمن.
    2. لإصدار شهادة الأمان المجانية (القفل الأخضر - HTTPS). بدونها، التحديثات الجديدة في تطبيقات أندرويد وآيفون **ترفض بشدة** الاتصال بسيرفر غير آمن (بدون HTTPS)، والـ HTTPS لا يعمل إلا بوجود دومين!.
    3. لكي يدخل المشرفون والمندوبون والدكاترة للوحة التحكم عبر رابط سهل التذكر.
